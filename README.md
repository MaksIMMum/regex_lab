# Лабораторна робота: Скінченні автомати та регулярні вирази за допомогою патерну State

## Опис проекту
У цьому проекті реалізовано скомпільований скінченний автомат (FSM) для розпізнавання регулярних виразів (Regex). Проект базується на об'єктно-орієнтованому патерні проектування **State** (Стан), де кожна частина регулярного виразу моделюється як окремий стан автомата, що відповідає за перевірку поточної позиції у вхідній стрічці та перехід до наступних станів.

### Підтримуваний функціонал
1. **Літерали**: розпізнавання літер англійської абетки (великих і малих, `a-z`, `A-Z`) та цифр (`0-9`).
2. **Символ точки (`.`)**: відповідає будь-якому одиничному символу.
3. **Оператор зірочки (`*`)**: відповідає нулю або більше повторенням попереднього стану.
4. **Оператор плюса (`+`)**: відповідає одному або більше повторенням попереднього стану.

## Пояснення імплементації та архітектури (State Pattern)

Архітектура побудована на взаємодії таких класів:

### 1. Базові класи станів (`State`, `StartState`, `TerminationState`)
* **`State`**: Абстрактний базовий клас для всіх станів. Містить метод `check_self(char)` для валідації символу та `check_next(char)` для здійснення переходу до наступних станів. Також динамічно ініціалізує список переходів `self.next_states = []` для кожного екземпляру, щоб уникнути проблеми розділеного стану (class-level list sharing).
* **`StartState`**: Початковий стан автомата, з якого починається перевірка будь-якої стрічки.
* **`TerminationState`**: Кінцевий приймаючий стан. Стрічка вважається успішно розпізнаною, якщо автомат зміг досягнути цього стану, повністю споживши вхідну стрічку.

### 2. Спеціалізовані стани
* **`AsciiState(symbol)`**: Стан для конкретного символу (літери або цифри). Повертає `True` у `check_self` лише за точного збігу.
* **`DotState`**: Відповідає за символ `.`. Повертає `True` для будь-якого символу у `check_self`.
* **`StarState(checking_state)`**: Стан для оператора Кліні (`*`). Обертає попередній стан (`checking_state`) і дозволяє як безпосередній перехід до нього, так і його повне пропускання (0 збігів).
* **`PlusState(checking_state)`**: Стан для оператора `+`. Потребує принаймні одного збігу з `checking_state`, після чого дозволяє циклічне повторення або перехід до наступних станів.

### 3. Компілятор FSM та симуляція (`RegexFSM`)
Клас `RegexFSM` приймає регулярний вираз на вхід і конструює ланцюг станів у конструкторі `__init__`.
Для розпізнавання стрічки використовується симуляція НКА (недетермінованого скінченного автомата) за допомогою пошуку в глибину (DFS) з мемоізацією (`memo`), що гарантує високу ефективність ($O(N \cdot M)$) та коректну роботу в разі недетермінізму та циклічних епсилон-переходів.

## Приклади роботи програми та результати

У файлі `regex.py` наведено приклад компіляції регулярного виразу `"a*4.+hi"` та перевірки кількох тестових рядків:

```python
if __name__ == "__main__":
    regex_pattern = "a*4.+hi"
    regex_compiled = RegexFSM(regex_pattern)

    print(regex_compiled.check_string("aaaaaa4uhi"))  # Має вивести: True
    print(regex_compiled.check_string("4uhi"))        # Має вивести: True
    print(regex_compiled.check_string("meow"))        # Має вивести: False
```

### Скріншот виводу виконання програми:

```text
True
True
False
```

### Додаткові тестові кейси
Під час розробки було проведено ретельне тестування для наступних регулярних виразів:
* `ab` на рядок `ab` -> **True**
* `a+` на рядок `aaaa` -> **True**
* `a+` на пустий рядок -> **False**
* `a*b+c` на рядок `bc` -> **True**
* `a.*c` на рядок `abbc` -> **True**
