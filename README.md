# Звіт з лабораторної роботи: Скінченні автомати та регулярні вирази за допомогою патерну State

## Опис проекту
У цьому проекті реалізовано скомпілований **Недетермінований скінченний автомат (НКА / NFA)** з підтримкою епсилон-переходів ($\varepsilon$-transitions) для розпізнавання регулярних виразів (Regex). 
Проект базується на поведінковому патерні проектування **State** (Стан), де кожна частина регулярного виразу моделюється як окремий стан автомата, що відповідає за валідацію символів та керування переходами до наступних станів.

### Підтримуваний функціонал
1. **Літерали (букви та цифри)**: точний збіг для англійських літер великого/малого регістру (`a-z`, `A-Z`) та цифр (`0-9`).
2. **Символ точки (`.`)**: відповідає будь-якому одиничному ASCII-символу.
3. **Оператор зірочки Кліні (`*`)**: відповідає нулю або більше повторенням попереднього стану.
4. **Оператор плюса (`+`)**: відповідає одному або більше повторенням попереднього стану.

## Пояснення імплементації та архітектури (State Pattern)

Архітектура проекту складається з взаємодії абстрактного базового класу та його конкретних станів-спадкоємців.

### 1. Ієрархія станів (Патерн State)
Всі стани успадковуються від абстрактного класу `State` та перевизначають його поведінку:
* **`State`**: Абстрактний базовий клас. Декларує обов'язкові методи `__init__` та `check_self(char)`. Надає загальну логіку переходів у методі `check_next`.
* **`StartState`**: Початковий стан автомата, який слугує вхідною точкою. Він не споживає символів вхідного рядка безпосередньо.
* **`TerminationState`**: Кінцевий приймаючий стан. Якщо після повного зчитування рядка автомат перебуває у цьому стані, рядок вважається розпізнаним.
* **`AsciiState`**: Стан для конкретного символу (літери або цифри). Повертає `True` у `check_self` за точного збігу з символом.
* **`DotState`**: Стан для wild-card символу `.`. Завжди повертає `True` для будь-якого символу.
* **`StarState`**: Стан для оператора `*`. Обертає стан, що повторюється, і налаштовує переходи так, щоб дозволити циклічний повтор або повний обхід (0 збігів).
* **`PlusState`**: Стан для оператора `+`. Схожий на `StarState`, але за допомогою логіки епсилон-замикання вимагає принаймні одного збігу, перш ніж дозволити перехід далі.

### 2. Симуляція автомата (NFA Simulation & $\varepsilon$-Closure)
Замість простого рекурсивного пошуку з поверненням (backtracking DFS), який може призвести до зависання на складних циклах, імплементовано **паралельну симуляцію НКА**:
* **Епсилон-замикання (`epsilon_closure`)**: Метод обчислює множину станів, у які можна перейти з поточної множини виключно за допомогою $\varepsilon$-переходів (без зчитування символу). Це дозволяє коректно обходити необов'язкові стани (як-от `StarState`) та виходити з циклів.
* **Паралельне просування станів**: Для кожного символу вхідної стрічки автомат знаходить усі можливі наступні стани для *всіх* поточних активних станів одночасно. Це робить процес розпізнавання лінійним за часом та виключає проблему нескінченної рекурсії.

## Інструкція з запуску

Для запуску проекту необхідна встановлена версія **Python 3.10** або новіша.

### Крок 1: Перевірка роботи автомата (regex.py)
Ви можете запустити файл `regex.py`, щоб виконати демонстраційні приклади:
```bash
python regex.py
```

### Крок 2: Запуск тестів (test.py)
Для запуску повної тестової бази у вас є дві опції:

1. **Запуск за допомогою вбудованого модуля `unittest`** (запустить 10 основних класичних тестів):
   ```bash
   python test.py
   ```

2. **Запуск за допомогою утиліти `pytest`** (запустить усі 63 розширені тестові випадки, включаючи параметризовані комбінації та граничні умови):
   ```bash
   pytest -v test.py
   ```
